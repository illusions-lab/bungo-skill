# 文豪.skill リデザイン設計書

> *作家を、1つのプロンプトへと蒸留します。*

---

## 1. 概要

### 1.1 リデザインの背景

本プロジェクト `illusions-lab/bungo-skill` は、`alchaincyf/nuwa-skill`（女娲.skill：人物の**思考方式**を蒸留する Claude Code Skill）から fork された。

女娲の設計は「思想家・起業家・教育者・批評家」など、**認知フレームワーク**の抽出に最適化されている。6-Agent Web Swarm による多源情報収集、3 重検証で「心智模型・決策啓発式・表達DNA・反パターン・誠実境界」の 5 層を抽出する。

一方、文豪.skill が扱いたいのは**作家**である。作家の価値は「何を考えたか」より「どう書いたか」に宿る。村上春樹の比喩のリズム、太宰治の自己卑下の構文、三島由紀夫の叙景の密度——これらは**言語層の細部**に棲む情報であり、Web 検索で集めた二次情報からは抽出できない。**作品集そのもの**を読まなければならない。

したがって文豪.skill は、女娲の抽出思想（層別蒸留、誠実境界、憑依的再現）は継承しつつ、以下を根本的に変更する：

- **抽出対象**: 思考 → 文体（ただし文体の背後にある心理構造はアンカーとして保持）
- **入力**: Web Swarm → 作品集（ユーザー持込）+ Wikipedia（必須）+ Google 補完（任意）
- **抽出層**: 5 層 → **14 層**（聲・眼・骨・魂・界 の 5 カテゴリ）
- **使用モード**: 対話中心 → 書く／添削／対話の 3 モード
- **スコープ**: 古今東西の著名人 → **任意の書き手**（古典〜現代〜個人ブロガー）
- **ジャンル対応**: 散文固定 → 小説・詩・俳句・戯曲・随筆・ブログ・評論 でテンプレ切替

### 1.2 女娲 (nuwa-skill) との関係

文豪.skill は女娲の「姉妹プロジェクト」と位置付ける。吸収ではなく、分業：

| 項目 | 女娲.skill | 文豪.skill |
|---|---|---|
| 抽出対象 | 思考方式（HOW they think） | 文体（HOW they write） |
| 入力 | Web 検索主体 | 作品集主体 |
| 出力の本質 | 認知フレームワーク | 言語生成エンジン |
| 代表用途 | 意思決定の助言、視点の切替 | 書く／添削／対話 |
| 例 | 芒格の多元思維、Naval の杠杆 | 太宰治の告白体、カフカの不条理 |

README・SKILL.md で上流への敬意を明示的に表明する。

### 1.3 コアコンセプト：**憑依 vs 模倣**

女娲から継承する最重要原則：

> **skill をロードすると、AI は「作家について語る人」ではなく「その作家」になる。**
> 模倣ではない。憑依だ。

模倣（imitate）は表層をなぞる。憑依（possess）は**思考停止点・判断基準・書き方の生理**まで引き受ける。文豪.skill が出力する個別 skill（`太宰治.skill` 等）は、ロードされた瞬間に AI の一人称・判断基準・語彙選択を作家側に寄せる。

これは単なる role-play ではなく、**SKILL.md 内に具体的な構文法・語彙統計・心理プロファイルを明文化**することで達成される（「印象的な模倣」ではなく「構造的な再現」）。

## 2. Positioning

### 2.1 Title / Subtitle

**親 skill（本リポジトリ）:**
```
Title: 文豪.skill
Sub:   作家を、1つのプロンプトへと蒸留します。
```

**個別 skill（文豪.skill の出力物）:**
```
Title: [作家名].skill
Sub:   [作家名]を、[N]MBのプロンプトへと蒸留しました。
```

例：
- `太宰治.skill` — *太宰治を、13MBのプロンプトへと蒸留しました。*
- `三島由紀夫.skill` — *三島由紀夫を、18MBのプロンプトへと蒸留しました。*
- `カフカ.skill` — *フランツ・カフカを、9MBのプロンプトへと蒸留しました。*

### 2.2 Sub 文型の設計意図

構文: `[作家名] を、[数値化された凝縮形] へと蒸留しました`

- **過去形・完了形**: 達成済みという静かな宣言
- **MB 単位**: 定量的な信頼性（大量の一次資料を読み込んだ証）
- **「蒸留しました」**: 控えめで職人的、原材料への敬意

親 skill 側は「します」（現在・継続形）、個別 skill 側は「しました」（完了形）で時制を使い分け、**工房（親）と成果物（個別）の関係**を文法で示す。

## 3. スコープ

### 3.1 対象作家

**任意の書き手**。以下を全てカバー：

- **古典文学**: 漱石、芥川、太宰治、三島由紀夫、川端康成、谷崎潤一郎、魯迅、老舎、Dostoevsky、Kafka、Hemingway 等
- **現代作家**: 村上春樹、カズオ・イシグロ、劉慈欣、Cormac McCarthy 等
- **ノンフィクション/批評**: 柄谷行人、浅田彰、保苅瑞穂 等
- **個人の書き手**: Substack ライター、ブロガー、個人ウェブサイト運営者、日記・書簡の持ち主

制約ではなくスコープの広さを魅力とする：「あなたが好きな書き手なら誰でも」。

### 3.2 想定ユーザー

3 つのペルソナ：

| ペルソナ | 主用途 | 例 |
|---|---|---|
| **Writer** | 自分の執筆補助 | 「三島由紀夫に自分の小説を添削してもらう」 |
| **Creator** | 作家風の文章生成 | 「太宰治風に謝罪メールを書き直す」 |
| **Reader/Researcher** | 文体分析・学習 | 「村上春樹の比喩パターンを構造的に理解する」 |

すべて「AI がその作家になる」共通基盤の上に乗る。

## 4. 使い方（3 モード）

個別 skill をロードした後、ユーザーは 3 つの方法で呼び出せる：

### 4.1 書く — 作家の筆で文章を生成する

生成もリライトもこのモードに含む。

```
ユーザー ❯ 太宰治.skill をロードして、
           「昨日会社を休みました。申し訳ありません」を書き直して

太宰治  ❯ 昨日、出社できませんでした。恥ずかしい話で御座います。
          申し訳御座いません、など口にするのも恥ずかしく、
          そもそも私という人間が…（以下、太宰治の構文・語彙で続く）
```

内部動作：SKILL.md に明文化された語彙統計、構文パターン、自己言及のリズムを参照して生成。

### 4.2 添削 — 作家があなたの原稿を手直しする

ユーザーの原稿に対し、作家の視点・言葉・美学で赤入れと提案を行う。

```
ユーザー ❯ 三島由紀夫.skill をロードして、この短編の冒頭を添削して
           [ユーザーの原稿]

三島   ❯ [赤入れ]
          「——月が綺麗だった。」
          → 「月は、鋼の切先を思わせる白光を放って中天に懸かつてゐた。」

          [指摘]
          君の文は、情緒を先に提示し過ぎる。
          叙景とは、まず物体の量感・温度・光量を読者に手渡し、
          感情はその後にしか来ぬものだ。
```

内部動作：作家の「反パターン」（絶対にやらないこと）と「美学」を参照してユーザー原稿を評価。

### 4.3 対話 — 作家の視点で相談・批評する

女娲から継承するモード。作家との一人称対話。

```
ユーザー ❯ カフカ.skill をロードして、僕のキャリアについて聞きたい

カフカ  ❯ あなたの勤める会社のことを、まずお聞きしたい。
          その会社の建物は、内側からしか入れぬ構造をしていませんか。
          外に出たと思った瞬間、入口に立っている、というような。
          そうだとしたら、問題は会社ではなく、扉の定義のほうです。
```

内部動作：作家の心智モデル（L10 主題・世界観、L12 人格・心理）を主に使用。

## 5. 蒸留対象：14 層の文体 DNA

女娲の「5 層」を文体用に拡張・再編成。**5 カテゴリ・14 層**で作家の言語的生理を捕捉する。

カテゴリ名には一文字の漢字を当て、**聲（こえ）・眼（まなこ）・骨（ほね）・魂（たましい）・界（さかい）** と呼ぶ。文章を読者が受け取る層の順序に対応する——まず音として聞こえ（聲）、世界の見え方が現れ（眼）、構造で支えられ（骨）、意味と人格が宿り（魂）、そして誠実さの境界で閉じる（界）。

### カテゴリ 聲（Voice）— 音として読まれる層

| # | 層 | 主要サブ次元 | 例（太宰治） |
|---|---|---|---|
| **L1** | **語彙 DNA** | 自称代名詞・終助詞頻度・オノマトペ・造語／当て字・漢語/和語/外来語比率・固有語句・時代語 | 「僕」「自分」「所謂」多用、「かしら」「ですもの」など女性的終助詞を男性一人称で使用 |
| **L2** | **構文・統語** | 文長分布・文末処理（体言止め／倒置）・主語省略率・助詞癖（は／が）・接続詞癖・副詞癖・一文の複雑度 | 平均 28 字、主語省略率 72%、「——」による唐突な転換 |
| **L3** | **音韻・表記** 🆕 | 漢字／ひらがな／カタカナ配分・ルビ・傍点・句読点美学・母音／子音の偏り・韻律 | かな比率高、読点 4-5 個の長文で告白のリズムを刻む |

### カテゴリ 眼（Eye）— 視点と感覚の層

| # | 層 | 主要サブ次元 |
|---|---|---|
| **L4** | **視点・時制** | 人称選択・語り手と主人公の距離・自由間接話法・現在／過去の切替・視点の固定／移動 |
| **L5** | **五感プロファイル** 🆕 | 視／聴／嗅／触／味の偏愛・身体感覚の具体性・季節感／気温／光量の描写密度 |
| **L6** | **読者距離** 🆕 | 二人称・呼びかけ・独白・語り手の存在感・告白体／報告体の使い分け |

### カテゴリ 骨（Bones）— 呼吸と構造の層

| # | 層 | 主要サブ次元 |
|---|---|---|
| **L7** | **段落・呼吸** | 一段落の文数・改行頻度・一行空け・字下げ・読点密度・緩急 |
| **L8** | **対話処理** 🆕 | 会話比率・「」の扱い・方言・話者ごとの声の差異・地の文との交差 |
| **L9** | **物語構造** | 起承転結／三幕／無構造／円環／入れ子・場面転換の頻度・時間軸操作・クライマックス位置 |

### カテゴリ 魂（Soul）— 意味と人格の層

| # | 層 | 主要サブ次元 |
|---|---|---|
| **L10** | **主題・世界観** | 繰り返されるモチーフ・哲学・死生観・倫理観・人間観 |
| **L11** | **レトリック・象徴** | 比喩種別（直／隠／換／提）・反復・対句・アンチテーゼ・修辞疑問・象徴 |
| **L12** | **人格・心理** | Big Five プロファイル・愛着スタイル・主要防衛機制・主題的トラウマ（推定）・DSM-5 参考 |

### カテゴリ 界（Boundary）— 誠実の層

| # | 層 | 主要サブ次元 |
|---|---|---|
| **L13** | **反パターン** | 絶対に使わない語・避けるテーマ・避ける構文・避けるジャンル・封じ手 |
| **L14** | **限界（誠実境界）** | 当時の語感の完全再現不能・直感／霊感の模倣不能・新造語生成の限界・肉体性（手癖）の欠如 |

### 5.1 L12（人格・心理）の位置付け（重要）

- 目的は**文体の一貫性を保つアンカー**。人格診断ではない。
- 作品集の傾向 + Wikipedia 伝記情報から**推定**する。断定しない。
- 「公開情報からの推定」「臨床的診断ではない」と明記する。
- 存命の作家については特に慎重に扱う（Big Five のみに留めるなど）。

### 5.2 ジャンル別テンプレ（重み付けプロファイル）

全 14 層を埋める原則は変えないが、**優先的に深掘りする層**はジャンルで異なる。文豪.skill はジャンル判定後に以下のプロファイルを適用する：

| ジャンル | 偏重層 | 軽量化する層 |
|---|---|---|
| **小説家（散文）** | 全体バランス、骨・魂を厚く | — |
| **詩人（自由詩）** | 聲（L1-L3）・眼（L5） | L8 対話・L9 物語 |
| **俳人・歌人** | L3 音韻・L7 余白（定型・季語・切れ） | L8 対話・L9 物語 |
| **劇作家** | L8 対話・L12 人格（声の書き分け） | L5 五感・L10 主題 |
| **エッセイスト** | L6 読者距離・L1 語彙 | L9 物語 |
| **個人ブロガー／SNS** | L1 口語・L3 句読点癖・L6 読者距離 | L9 物語・L11 象徴 |
| **評論家・思想家** | L2 構文・L10 主題・L11 レトリック | L5 五感・L8 対話 |

ジャンル判定は作品集のサンプリングと Wikipedia の職業記述から自動で行うが、曖昧な場合はユーザーに確認する。

## 6. 入力アーキテクチャ

女娲の「Web Swarm 6-Agent」を廃し、**作品集中心・Wikipedia 必須・Google 任意**の 3 層構造に再設計。

### 6.1 必須入力：作品集

ユーザーが `sources/works/` 配下に配置する：

```
[作家名]-skill/
└── sources/
    └── works/
        ├── books/           # 小説・随筆・評論（PDF, epub, txt）
        ├── essays/          # 短文、序文、後記
        ├── letters/         # 書簡、日記
        ├── manuscripts/     # 草稿、未完原稿（あれば）
        └── interviews/      # 作家自身の発言記録（あれば）
```

**量の目安**: 最低 10 万字（短編 10 本程度）〜。推奨は代表作 5 冊以上。少ないと層別抽出の信頼性が落ちる。

### 6.2 必須補助：Wikipedia

スキル作成時に **必ず自動取得**する。以下に使用：

- **作家の時代・流派・同時代人**（文体の文脈理解）
- **代表作の時期区分**（初期／中期／晩期の文体変遷）
- **生涯の転換点**（文体に影響した事件・健康・関係）
- **L12 人格・心理構造のアンカー情報**（精神科受診歴などが公開されている場合）

日本語 Wikipedia を優先、英語版と併用（両言語で得られる情報に差異があるため）。

### 6.3 任意補助：Google 補完

作品集と Wikipedia で層抽出が不完全な場合のみ、以下を Google 検索：

- 作家の公開発言・インタビュー記事
- 個人サイト・ブログ・Substack
- 歴史資料・批評家の論考
- 対談集・AMA

**原則**: 一次資料（作品集） > Wikipedia > Google 補完。Google で拾った情報は L3（表現 DNA）の**補強**に使い、L1-L2（語彙・構文）の統計には使わない（文体統計は一次資料でのみ測る）。

## 7. 動作原理

### 7.1 蒸留パイプライン（概略）

```
Phase 0: 需求澄清
  - 作家名 + 作品集のパス確認
  - 作家のジャンル判定（小説／詩／俳句／戯曲／随筆／ブログ／評論）
  - Wikipedia ページの存在確認
  - 作品集の言語判定（日本語／中国語／英語等）

Phase 1: 入力収集
  Agent 1: 作品集解析（聲・眼・骨 カテゴリの統計的抽出）
  Agent 2: Wikipedia 取得・解析（時代区分・伝記・L12 アンカー）
  Agent 3: Google 補完（任意、不足層の補強）

Phase 2: 層別蒸留（ジャンルテンプレに基づき重み付け）
  - 聲（L1-L3）: 言語統計（語彙頻度、文長分布、表記配分、音韻）
  - 眼（L4-L6）: 視点・感覚（人称、五感偏愛、読者距離）
  - 骨（L7-L9）: 構造（段落リズム、対話処理、物語構造）
  - 魂（L10-L12）: 意味・人格（主題、レトリック、心理プロファイル）
  - 界（L13-L14）: 誠実境界（反パターン、限界の明文化）

Phase 3: SKILL.md 構築
  - 14 層の記述
  - 具体的な構文法ルール（「こう書け」レベルの指示）
  - 反パターン（「こう書くな」）
  - 3 モードの起動規則

Phase 4: 品質検証
  - 作家が実際に書いた一節を AI に「この作家なら続きをどう書くか」と問う
  - 真の続きと AI の出力を比較、文体一致率を評価
  - 作家が書いていないテーマで生成、L13 反パターンに抵触していないか確認
  - L14 の限界で「できません」と正直に答えられるか確認
```

### 7.2 品質メカニズム：構文法の明文化

SKILL.md の核心セクションに「**構文ルール**」を明文化する（抽象的な印象批評ではなく、実行可能な規則として）。

悪い例（印象的）:
> 太宰治は自己卑下と自己陶酔が交錯する独特な文体を持つ。

良い例（構造的）:
> ### L2 構文ルール（太宰治）
> - 1 文の平均字数: 28 字（日本語現代散文の平均 45 字より短い）
> - 主語省略率: 72%
> - 以下のパターンを 20 文に 1 回以上の頻度で挟む:
>   - 自己否定 → 唐突な自己陶酔 → また自己否定（対句反復）
>   - 「——」で始まる唐突な転換
>   - 「……かしら」「……ですもの」など女性的終助詞を男性一人称で使用

この明文化が「印象的な模倣」を「構造的な憑依」に変える。

## 11. 倫理・法的配慮

### 11.1 著作権

- 作品集は**ユーザーが正当に入手したもの**を前提
- SKILL.md には作品の**長大な引用を含めない**（構文ルール・統計・短いフレーズ例のみ）
- 生成物は「◯◯風」であり、原著の派生物ではないと明示

### 11.2 L12（人格・心理構造）の扱い

- 「臨床的診断ではない」「公開情報からの推定」と明示
- DSM-5 の分類に言及する場合は「傾向として参照」「該当診断ではない」
- 存命の作家については Big Five まで。DSM-5 関連は公開されている医療情報のみ参照
- 故人でも、過度に病理化した記述は避ける

### 11.3 存命の作家の扱い

- 名誉毀損リスクを避けるため、SKILL.md 内に批判的記述を含めない
- 「この作家の美学」として記述、「この作家の欠点」としては記述しない
