# README 大改造：義肢メタファー + 失敗展示室（A + D 合体）

> Codex round 1 レビューを経て全面改訂（2026-04-19）。改訂の要約は末尾「Review iterations」を参照。

## Goal

`/Users/iktahana/bungo-skill/README.md`（現在 **124 行**）を再設計する。2 つの方針を同時に適用：

- **Option A（保守的）**：「憑依」→「義肢」メタファーへの差し替え、末尾に「反対尋問」節を追加。
- **Option D（最ラディカル）**：冒頭のヒーロー位置に「失敗展示室」（素の AI 声 vs skill 適用 vs L13 反パターン）を置く。

**D が読者の最初の衝撃を担い、A が論理骨格を支える**。

## Why both

- **D だけ**なら作例集に見え、install / 原理 / 敵対テスト言及が押し込まれる。
- **A だけ**なら既存 README の延長で、「nuwa の fork」感が抜けきらない（hero 行に `Fork of nuwa-skill` badge が現存）。
- **D + A** なら、冒頭で「skill が何を**しない**か」を示し、本体で「何を**する**か」を説明できる。義肢メタファーが橋を架ける——義肢は「欠損」を前提にするため、「何を補うか」が明確になる。

## Non-goals

- **`SKILL.md` / `references/skill-template.md` の用語変更は本スコープ外**。両ファイルは現在も「憑依」を核に据えており（`SKILL.md` L11-L16、`skill-template.md` L10-L25）、README を先行して「義肢」に差し替えると一時的に repo 全体で用語が分裂する。本計画では README に明示的な注記を置いて「これは README の言い換えであり、工房側の terminology alignment は後続タスク」と断る（R4 対応）。
- `dazai-skill` など子 skill への波及は別スレッド。
- ロゴ・画像追加なし、テキストのみ。
- i18n なし（`README_*.md` は削除済み、日本語単独維持）。
