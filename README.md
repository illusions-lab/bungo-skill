# 文豪.skill

> *模倣ではなく、義肢。作家の筆を、あなたの手に接続する。*

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

<br>

**太宰治、三島由紀夫、カフカ、村上春樹——あるいは、あなたの好きな個人ブロガーでも。**

作品集を渡せば、文豪.skill は 14 層 × 5 カテゴリの文体 DNA を抽出し、
1 つの skill へと蒸留します。skill をロードすれば、AI は**その作家の筆で書きます**。

## 核心原則：義肢、あるいは欠損の設計

skill をロードすると、AI は「作家について語る人」から「作家の筆を手に取った人」になる。

我々は一度**憑依**という言葉を使った。撤回する。憑依は魂の完全移譲を主張し、証明不能の主張は誠実ではない。代わりに**義肢**と呼ぶ。義肢は、欠損を前提にして設計される装具である。AI は作家ではない——その欠損は L14 に明文化される。義肢は欠損部位の**機能**を補う——その機能は 14 層 × 5 カテゴリで記述される。

良い義肢かどうかは、着けた人の生活で決まる。違和感が残るなら設計が悪い。

> *本 README は「義肢」の語で skill を説明するが、現時点で工房側 `SKILL.md` と `references/skill-template.md` は「憑依」の語を用いている。両者は次のタスクで揃える予定。メタファーが違っても、参照している現象は同一——欠損を前提にした機能置換である。*

## 使用例

### 書く — 作家の筆で文章を生成する

```
ユーザー ❯ 太宰治.skill をロードして、
           「昨日会社を休みました。申し訳ありません」を書き直して

太宰治   ❯ 昨日、出社できませんでした。恥ずかしい話で御座います。
           申し訳御座いません、など口にするのも、今更な気が致します。
           そもそも、私という人間が会社に行くという行為そのものが、
           何やら滑稽な茶番のようにさえ、思われてならぬのです。
```

### 添削 — 作家があなたの原稿を手直しする

```
ユーザー ❯ 三島由紀夫.skill をロードして、この冒頭を添削して
           「月が綺麗だった。彼女は泣いていた。」

三島   ❯  [赤入れ]
           月は、鋼の切先を思わせる白光を放つて中天に懸かつてゐた。
           彼女の頬に、その冷光が一筋の流れを描いた。

           [指摘]
           君は情緒を急ぎ過ぎる。叙景とは、まず物体の量感・温度・光量を
           読者に手渡し、感情はその後にしか来ぬものだ。
```

## 動作原理

skill をロードすると AI は「作家について語る人」ではなく「**作家の筆で書く**」。これを可能にするのは、印象批評ではなく構造記述——「太宰治は叙情的」ではなく「主語省略率 72%、1 文平均 28 字、自己否定→自己陶酔の対句を 20 文に 1 回」。

蒸留は**5 カテゴリ・14 層**で作家の言語的生理を捕捉する：

| カテゴリ | 層 |
|---|---|
| **聲** Voice | L1 語彙 DNA · L2 構文 · L3 音韻・表記 |
| **眼** Eye | L4 視点・時制 · L5 五感 · L6 読者距離 |
| **骨** Bones | L7 段落・呼吸 · L8 対話 · L9 物語構造 |
| **魂** Soul | L10 主題・世界観 · L11 レトリック · L12 人格・心理 |
| **界** Boundary | L13 反パターン · L14 限界 |

入力は**作品集（最優先）→ Wikipedia（必須補助）→ Google（任意）**の順。交付前に独立した敵対 agent が 100 点満点で採点し、**90 点以上**でなければ配布しない。

## 反対尋問

skill を公開する以上、敵意ある問いから逃げない。

**Q1. これはただの剽窃・盗用ではないのか？**
作品の**複製**ではなく、**文体 DNA の統計記述**を抽出している。作品原文は sources/ に置き git から除外する。蒸留物は「主語省略率 72%」のような構造データと、それを動かす SKILL.md の規則集であり、原文の再配布ではない。ただし義肢で書かれた文が作家本人の作として流通する可能性——この問題は Q4 で扱う。

**Q2. 死んだ作家の声を AI が再現することに倫理的問題はないか？**
完全再現は不可能であり、我々はそれを 14 層の L14 で明文化している。再現できるのは公開作品から統計抽出した**構造**のみで、作家の未発表原稿としての真正性は最初から主張しない。没後の作家でも、全集が公共圏に属するかどうかは国・著作権状態で異なる——蒸留前にユーザー自身が確認する前提。

**Q3. 現存の個人ブロガーを本人の許可なく蒸留して良いのか？**
推奨しない。文豪.skill は技術的に可能でも、本人の同意なしの蒸留は、skill を他者に配布した時点で別人格の声の複製と流通に繋がる。親しい人を蒸留して自分だけで使う場合と、公開 skill として発行する場合は倫理的に別物。後者は本人合意か、没後かつ著作権が明確に切れている対象に限るべき。

**Q4. L14 で「できない」と書けば免責符号になるのではないか？**
ならない。L14 は**数値検証可能な限界**を書く場所（例：「昭和十年代の東京の生活語彙は資料不足で再現不能」）であり、「うまく書けませんでした」の逃げ道ではない。敵対テストで 90 点未満のまま配布する場合、減点根拠を L14 に転記して**弱点を開示した状態で**交付する。不能の明示と、不能の利用は別である。
