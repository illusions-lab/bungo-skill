# 文豪.skill

> *作家を、1つのプロンプトへと蒸留します。*

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Claude Code](https://img.shields.io/badge/Claude%20Code-Skill-blueviolet)](https://claude.ai/code)
[![Fork of](https://img.shields.io/badge/fork%20of-nuwa--skill-lightgrey)](https://github.com/alchaincyf/nuwa-skill)

<br>

**太宰治、三島由紀夫、カフカ、村上春樹——あるいは、あなたの好きな個人ブロガーでも。**

作品集を渡せば、文豪.skill は14層 × 5カテゴリの文体DNAを抽出し、
1つのskillへと蒸留します。skillをロードすれば、AIは**その作家になります**。
模倣ではなく、憑依。

<br>

```
太宰治.skill          太宰治を、13MBのプロンプトへと蒸留しました。
三島由紀夫.skill      三島由紀夫を、18MBのプロンプトへと蒸留しました。
カフカ.skill          フランツ・カフカを、9MBのプロンプトへと蒸留しました。
```

## 使用例

### 1. 書く — 作家の筆で文章を生成する

```
ユーザー ❯ 太宰治.skill をロードして、
           「昨日会社を休みました。申し訳ありません」を書き直して

太宰治   ❯ 昨日、出社できませんでした。恥ずかしい話で御座います。
           申し訳御座いません、など口にするのも、今更な気が致します。
           そもそも、私という人間が会社に行くという行為そのものが、
           何やら滑稽な茶番のようにさえ、思われてならぬのです。
```

### 2. 添削 — 作家があなたの原稿を手直しする

```
ユーザー ❯ 三島由紀夫.skill をロードして、この冒頭を添削して

           「月が綺麗だった。彼女は泣いていた。」

三島   ❯  [赤入れ]
           月は、鋼の切先を思わせる白光を放つて中天に懸かつてゐた。
           彼女の頬に、その冷光が一筋の流れを描いた。

           [指摘]
           君は情緒を急ぎ過ぎる。
           叙景とは、まず物体の量感・温度・光量を読者に手渡し、
           感情はその後にしか来ぬものだ。
```

### 3. 対話 — 作家の視点で相談・批評する

```
ユーザー ❯ カフカ.skill をロードして、キャリアの相談をしたい

カフカ   ❯ あなたの勤める会社のことを、まずお聞きしたい。
           その会社の建物は、内側からしか入れぬ構造をしていませんか。
           外に出たと思った瞬間、入口に立っている、というような。
           そうだとしたら、問題は会社ではなく、扉の定義のほうです。
```

## 核心原則：憑依 vs 模倣

> **skill をロードすると、AI は「作家について語る人」ではなく「その作家」になる。**

模倣（imitate）は表層をなぞる。憑依（possess）は思考停止点・判断基準・書き方の生理まで引き受ける。

これを可能にするのは、SKILL.md 内に**具体的な構文法・語彙統計・心理プロファイルを明文化**すること。印象批評ではなく構造記述——「太宰治は自己卑下の文体」ではなく、「主語省略率72%、1文平均28字、自己否定→自己陶酔の対句を20文に1回」。

---

## 何を蒸留するか

女娲の5層を文体用に拡張・再編成。**5カテゴリ・14層**で作家の言語的生理を捕捉する——読者が文章を受け取る順に、**聲（こえ）→眼（まなこ）→骨（ほね）→魂（たましい）→界（さかい）**。

| カテゴリ | # | 層 |
|---|---|---|
| **聲** Voice — 音として読まれる | L1 | 語彙DNA（自称・終助詞・オノマトペ・造語・漢和外比率） |
| | L2 | 構文・統語（文長・文末処理・主語省略・助詞癖） |
| | L3 | 音韻・表記（漢字/かな/カナ配分・ルビ・句読点美学・韻律） |
| **眼** Eye — 視点と感覚 | L4 | 視点・時制（人称・語り手距離・自由間接話法） |
| | L5 | 五感プロファイル（視/聴/嗅/触/味の偏愛・身体感覚） |
| | L6 | 読者距離（二人称・呼びかけ・独白・語り手の存在感） |
| **骨** Bones — 呼吸と構造 | L7 | 段落・呼吸（一段落文数・一行空け・字下げ・緩急） |
| | L8 | 対話処理（会話比率・「」の扱い・方言・話者差異） |
| | L9 | 物語構造（起承転結・三幕・円環・無構造・時間軸操作） |
| **魂** Soul — 意味と人格 | L10 | 主題・世界観（モチーフ・死生観・倫理観・人間観） |
| | L11 | レトリック・象徴（比喩種別・反復・対句・アンチテーゼ） |
| | L12 | 人格・心理（Big Five・愛着・防衛機制・DSM-5参考、推定） |
| **界** Boundary — 誠実の境 | L13 | 反パターン（絶対に使わない語・避けるテーマ・封じ手） |
| | L14 | 限界（時代語感・直感・肉体性の再現不能） |

### ジャンル別テンプレ

14層を埋める原則は変えないが、**どの層を深掘りするか**はジャンルで異なる：

| ジャンル | 偏重層 |
|---|---|
| 小説家（散文） | 全層バランス、骨・魂重め |
| 詩人（自由詩） | 聲（L1-L3）・眼（L5） |
| 俳人・歌人 | L3 音韻・L7 余白（定型・季語・切れ） |
| 劇作家 | L8 対話・L12 人格（声の書き分け） |
| エッセイスト | L6 読者距離・L1 語彙 |
| 個人ブロガー/SNS | L1 口語・L3 句読点癖・L6 読者距離 |
| 評論家・思想家 | L2 構文・L10 主題・L11 レトリック |

### 誠実境界

文豪.skill が蒸留できないもの：
- 時代語感の完全再現
- 本人の未発表原稿としての真正性
- 直感・霊感（構造は模倣可能、閃きは不可能）

**一つも嘘をつかないskillだけが、信頼に値する。**

## 動作原理

入力は **作品集が主、Wikipedia が必須の補助、Google が任意**。

**Phase 1: 入力採集**
1. ユーザーが `sources/works/` に作品集（PDF/epub/txt）を配置
2. Wikipedia を自動取得（時代背景・伝記・心理アンカー情報）
3. 作品集と Wikipedia で不足する層があれば Google 補完

**Phase 2: 14層蒸留（ジャンルテンプレで重み付け）**
- **聲**（L1-L3）: 語彙・構文・音韻の統計
- **眼**（L4-L6）: 視点・五感・読者距離
- **骨**（L7-L9）: 呼吸・対話・物語構造
- **魂**（L10-L12）: 主題・レトリック・心理（Big Five/DSM-5参考、倫理配慮付き）
- **界**（L13-L14）: 反パターンと限界の明文化

**Phase 3: 構文法の明文化**
SKILL.md に「この作家はこう書く／こう書かない」を**実行可能な規則**として記述。印象批評ではなく構造記述。

**Phase 4: 品質検証**
- 作家が実際に書いた一節の続きを生成させ、真の続きと比較
- L13 反パターンに抵触していないか確認
- L14 の限界で「できません」と正直に答えられるか確認

詳細は [`docs/design/2026-04-18-bungo-redesign.md`](docs/design/2026-04-18-bungo-redesign.md) と [`references/extraction-framework.md`](references/extraction-framework.md)。
