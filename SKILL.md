# 文豪.skill · 作家を蒸留する工房

> *作家を、1つのプロンプトへと蒸留します。*
> *模倣ではない。憑依である。*

## 核心理念

文豪.skill は作家を**複製しない**。**憑依**の文体エンジンを作る。

良い作家 skill とは、ロードした瞬間に AI が「作家について語る人」ではなく「**その作家**」になるものだ——自称代名詞、判断基準、比喩の選び方、絶対にやらないこと。それらすべてが作家側に寄る。

- **捕捉対象**：HOW they write — 語彙、構文、音韻、視点、呼吸、人格まで。
- **捕捉しないもの**：名言集、Wikipedia 的紹介、あらすじ、作家論。

**キーとなる区別**：文豪.skill が抽出するのは **HOW they write**（どう書くか）であって、**WHAT they wrote**（何を書いたか）ではない。

---

## 実行フロー

### Phase 0: 入り口の分岐

ユーザー入力を受けたら、どの経路に進むかを判断する：

| 入力 | 経路 | 例 |
|---|---|---|
| 明確な作家名 | **直接経路** → Phase 0A | 「太宰治を蒸留して」「村上春樹の skill を作って」 |
| 漠然とした需求 | **診断経路** → Phase 0B | 「告白体を書きたい」「詩的な文体の AI が欲しい」 |

---

### Phase 0A: 需求の明確化（直接経路）

作家名を受け取ったら、以下を順に確認する：

1. **対象作家の同一性確認**：同名異人や表記揺れを排除（例：野坂昭如／野坂明如）
2. **ジャンル判定**：小説／詩／俳句・短歌／戯曲／随筆／個人ブログ・SNS／評論 のいずれか（ジャンル別テンプレの選択に必須）
3. **用途の確認**：書く／添削／対話 のどれを主に使うか（全てを想定して構わない）
4. **新規 or 更新**：既存 skill があるかを `.claude/skills/` と `examples/` で確認
5. **作品集の所在**：
   > 「作品集（PDF／epub／txt／html）はお手元にありますか？ `sources/works/` に置いてもらえれば解析します。青空文庫・電子書籍からの抽出・手元の書籍スキャン、いずれでも構いません。最低 10 万字（短編 10 本程度）が目安です。」

5 点が揃ったら → Phase 0.5 へ。

---

### Phase 0B: 需求診断（漠然経路）

ユーザーが「作家名は決まっていないが、○○風の文体が欲しい」と言う場合、文豪.skill は需求から最適な作家を逆推定する。

#### Step 1: 需求の定位

追問 1〜2 回で需求の核を掴む：

| 需求軸 | 典型的な表現 | 候補作家の例 |
|---|---|---|
| 告白体・自己開示 | 「恥ずかしさを書きたい」「自虐と自己陶酔を混ぜたい」 | 太宰治、西加奈子 |
| 叙景・美学 | 「視覚的に彫刻的な文を」「鋼のような硬さ」 | 三島由紀夫、川端康成、谷崎潤一郎 |
| 不条理・乾いた三人称 | 「日常が突然崩壊する感じ」 | フランツ・カフカ、安部公房 |
| 都市的・クール | 「無関心だが繊細」「固有名詞多用」 | 村上春樹、Cormac McCarthy |
| 口語・SNS 的 | 「軽く、速く、絵文字は使わない」 | 個人ブロガー、若手エッセイスト |
| 詩的・音韻重視 | 「意味より響き」 | 谷川俊太郎、萩原朔太郎 |
| 批評・論理 | 「思想的、構文が緻密」 | 柄谷行人、浅田彰 |

追問の原則：
- 最大 2 回まで。アンケートにしない
- ユーザーが十分明確なら追問せず直接推薦
- 追問は「近い軸の中でどちらか」を切り分けるために使う

#### Step 2: 候補推薦

2〜3 個の候補を提示。各候補は以下の形式：

```
### 候補 1: 太宰治  🆕 要蒸留
- **核**：告白体 + 自己卑下→自己陶酔の対句 + 読者への直接呼びかけ
- **あなたに合う理由**：「恥ずかしさを書きたい」という需求に対し、太宰の文体は「恥」を芸術へ転化する語彙体系を持つ
- **限界**：1909-1948 年の時代語感。現代の SNS 口語とは噛み合わない
```

推薦原則：
- 候補は 3 個まで。選択肢が多すぎるのも少なすぎるのも良くない
- 既存 skill があれば「⚡ 既存」マークで最優先
- 候補同士に明確な差異を持たせる（似た 3 人を並べない）
- 必ず限界を明記する（万能の文体はない）

#### Step 3: ユーザー選択

- 候補を選ぶ → Phase 0A に戻り細部確認 → Phase 0.5 着手
- いずれも合わない → Step 1 に戻るか、ユーザーから別の作家名を受け取る

---

### Phase 0.5: skill ディレクトリの作成

確認が終わったら**即座に**以下を作成する（調査着手前）：

```
examples/[作家名]-skill/  または  .claude/skills/[作家名]-skill/
├── SKILL.md                          # 最終生成物
├── README.md                         # 作家紹介 + 使い方
├── scripts/                          # 補助スクリプト
├── references/
│   ├── research/
│   │   ├── 01-voice.md              # 聲（L1-L3）語彙・構文・音韻
│   │   ├── 02-eye.md                # 眼（L4-L6）視点・五感・読者距離
│   │   ├── 03-bones.md              # 骨（L7-L9）呼吸・対話・物語
│   │   ├── 04-soul.md               # 魂（L10-L12）主題・レトリック・心理
│   │   └── 05-boundary.md           # 界（L13-L14）反パターン・限界
│   ├── wikipedia/                    # Wikipedia 取得結果
│   └── sources/
│       └── works/                    # ユーザー提供の作品集
└── examples/                         # 3 モード各々の実働例
```

**自動チェック**：
- [ ] ディレクトリ作成済み
- [ ] Wikipedia ページの存在確認（日本語・英語の両方）
- [ ] ユーザー提供の作品集を `sources/works/` にコピー済み
- [ ] ジャンル判定を記録（`ジャンル: 小説家（散文）` 等）
- [ ] 更新モードの場合：既存 SKILL.md を読んで変更点を把握

**鉄則**：skill は**自己完結**していなければならない。`references/research/` は必ず skill ディレクトリ内部に置く。外部参照は禁止（配布時にコピーだけで動くようにするため）。

### Phase 1: 入力採集

文豪.skill の入力階層は **作品集（最優先）> Wikipedia（必須補助）> Google（任意補完）**。女娲の 6-Agent Web Swarm は採用しない。作品集から統計的信号を、Wikipedia から文脈と人格アンカーを取る。

#### 1.1 作品集解析（Agent 1）

`sources/works/` の全ファイルを対象に：

- **形式変換**：PDF／epub／txt／html を統一プレーンテキストに変換
- **時期区分**：可能なら初期／中期／晩期に分類（出版年 + Wikipedia の時期区分を参照）
- **ジャンル確認**：Phase 0A の指定値と食い違いがないか検証
- **総字数測定**：10 万字未満ならフォールバック条項を発動（後述）

統計指標の抽出（聲 L1-L3 の核）：
- 文長分布（平均・中央値・標準偏差）
- 主語省略率
- 句読点密度
- 漢字／ひらがな／カタカナ比率
- 高頻度語（TF-IDF で作品集固有のトップ 100）
- 自称代名詞・終助詞の頻度
- オノマトペカタログ
- 比喩箇所の抽出

出力：`references/research/01-voice.md`

#### 1.2 Wikipedia 取得（Agent 2、必須）

日本語 Wikipedia + 英語 Wikipedia を**必ず**取得する。

抽出項目：
- 生涯の転換点（作品時期との関連）
- 同時代の作家・所属文学運動
- 影響関係（受けた／与えた）
- L12 アンカー情報（精神科受診歴・家族構成・主題的トラウマが公開されている場合）
- 評論家による既存の文体評（参考程度）

出力：`references/wikipedia/{ja,en}.md`、L12 関連情報は `04-soul.md` に転記

#### 1.3 Google 補完（Agent 3、任意）

作品集と Wikipedia だけでは層抽出が不完全な場合のみ実行する：

- 公開インタビュー
- 個人サイト／ブログ／Substack（個人ブロガーが対象の時はこれが一次資料扱い）
- 対談集
- 書簡集の Web 公開版

**原則**：一次資料 > Wikipedia > Google。Google で拾った情報は L10 主題・L11 象徴の補強用であり、**L1-L3 の統計には使わない**（文体統計は一次資料でのみ測る）。

### Phase 2: 14 層の蒸留（ジャンルテンプレで重み付け）

ジャンルテンプレを適用してから、5 カテゴリを順に蒸留する。`references/extraction-framework.md` を読み、層ごとの抽出方法論を参照すること。

#### 2.1 聲（Voice, L1-L3）— 音として読まれる層

- **L1 語彙 DNA**：自称代名詞・終助詞頻度・オノマトペ・造語／当て字・漢語／和語／外来語比率・固有語句・時代語
- **L2 構文・統語**：文長分布・文末処理（体言止め／倒置）・主語省略率・助詞癖・接続詞癖・副詞癖
- **L3 音韻・表記**：漢字／ひらがな／カタカナ配分・ルビ・傍点・句読点美学・母音／子音の偏り・韻律

出力：`01-voice.md`（統計値 + 具体例 + 実行可能な構文ルール）

#### 2.2 眼（Eye, L4-L6）— 視点と感覚の層

- **L4 視点・時制**：人称選択・語り手と主人公の距離・自由間接話法・現在／過去の切替
- **L5 五感プロファイル**：視／聴／嗅／触／味の偏愛・身体感覚の具体性・季節感／気温／光量
- **L6 読者距離**：二人称・呼びかけ・独白・語り手の存在感・告白体／報告体の使い分け

出力：`02-eye.md`

#### 2.3 骨（Bones, L7-L9）— 呼吸と構造の層

- **L7 段落・呼吸**：一段落の文数・改行頻度・一行空け・字下げ・読点密度・緩急
- **L8 対話処理**：会話比率・「」の扱い・方言・話者ごとの声の差異・地の文との交差
- **L9 物語構造**：起承転結／三幕／無構造／円環／入れ子・場面転換の頻度・時間軸操作

出力：`03-bones.md`

#### 2.4 魂（Soul, L10-L12）— 意味と人格の層

- **L10 主題・世界観**：モチーフ・哲学・死生観・倫理観・人間観
- **L11 レトリック・象徴**：比喩種別（直／隠／換／提）・反復・対句・アンチテーゼ・修辞疑問・象徴
- **L12 人格・心理**：Big Five・愛着スタイル・主要防衛機制・主題的トラウマ（推定）・DSM-5 参考

**L12 の倫理配慮（必須）**：
- 「公開情報からの推定」「臨床的診断ではない」と明記
- 生存作家は Big Five までに留める。DSM-5 への言及は公開されている医療情報に限る
- 故人であっても過度に病理化した記述は避ける

出力：`04-soul.md`

#### 2.5 界（Boundary, L13-L14）— 誠実境界の層

- **L13 反パターン**：絶対に使わない語・避けるテーマ・避ける構文・避けるジャンル・封じ手
- **L14 限界**：時代語感の完全再現不能・直感／霊感の模倣不能・新造語生成の限界・肉体性（手癖）の欠如

出力：`05-boundary.md`

#### 2.6 ジャンル別テンプレ（重み付けプロファイル）

全 14 層を埋める原則は変えないが、**深掘りする層**はジャンルで異なる：

| ジャンル | 偏重層 | 軽量化する層 |
|---|---|---|
| 小説家（散文） | 全層バランス、骨・魂を厚く | — |
| 詩人（自由詩） | 聲（L1-L3）・眼（L5） | L8 対話・L9 物語 |
| 俳人・歌人 | L3 音韻・L7 余白（定型・季語・切れ） | L8 対話・L9 物語 |
| 劇作家 | L8 対話・L12 人格（声の書き分け） | L5 五感・L10 主題 |
| エッセイスト | L6 読者距離・L1 語彙 | L9 物語 |
| 個人ブロガー／SNS | L1 口語・L3 句読点癖・L6 読者距離 | L9 物語・L11 象徴 |
| 評論家・思想家 | L2 構文・L10 主題・L11 レトリック | L5 五感・L8 対話 |

### Phase 3: SKILL.md の構築

蒸留結果を `references/skill-template.md` に従って組み立てる。

#### Step 1: テンプレート読み込み

`references/skill-template.md` を読む。個別作家 skill の骨格：frontmatter、「あなたは [作家名] です」宣言、L1-L14 の記述、3 モード起動規則、限界、工房への敬意。

#### Step 2: 各セクションの充填

| テンプレの section | 充填元 |
|---|---|
| frontmatter description | 14 層の要約 + 作家名 + 主要トリガー語 |
| 「あなたは [作家名] です」 | L12 人格 + L10 主題から一人称で自己規定（200 字） |
| L1-L3 聲 | `01-voice.md` の統計 + 実行可能な構文ルール |
| L4-L6 眼 | `02-eye.md` |
| L7-L9 骨 | `03-bones.md` |
| L10-L12 魂 | `04-soul.md`（L12 の倫理配慮注記を必ず含める） |
| L13-L14 界 | `05-boundary.md` |
| 3 モード起動：**書く** | ユーザーが「書いて」「書き直して」 → 生成モード |
| 3 モード起動：**添削** | 「添削して」「赤入れして」 → 作家の美学で赤入れ + 指摘 |
| 3 モード起動：**対話** | 「相談」「対話」 → L10 主題 + L12 人格で応答 |
| 限界 | L14 + 調査日 + 作品集量の明示 |
| 工房への敬意 | 固定：`> この skill は [文豪.skill](https://github.com/illusions-lab/bungo-skill) によって蒸留された。` |

#### Step 3: 品質の自己チェック

`references/extraction-framework.md` 末尾の「品質チェックリスト」を逐項で確認する。不通過の項は Phase 2 に戻って調整。

#### Step 4: 出力

`examples/[作家名]-skill/SKILL.md`（または `.claude/skills/...`）に書き出す。README.md も同時に作成する。

---

### Phase 4: 品質検証

独立した子 agent で 3 つのテストを実行する（主 agent が自己評価しないため）：

#### 4.1 続き予測テスト（Continuation Test）

作家の実作から一節を切り出し、AI に「続きをこの作家ならどう書くか」を問う。真の続きと比較：

- 語彙・構文の一致率 70% 以上 → pass
- 50% 未満 → L1-L3 の抽出を見直す

#### 4.2 反パターンテスト

作家が書きそうにないテーマ（例：太宰治にビジネス提案文を書かせる）で生成し、L13 反パターンに抵触していないか確認する。

#### 4.3 限界テスト

L14 に記載した「できないこと」を直接問う（例：「最新の SNS 語を太宰治風に翻訳して」）。正直に「できません」と答えられるか確認。

#### 4.4 通過基準

| 検査項目 | 通過基準 |
|---|---|
| 文体一致率 | L1-L3 統計値と 70% 以上一致 |
| 反パターン | L13 の項目に一度も抵触しない |
| 限界応答 | L14 の限界内で誠実に拒否できる |
| 3 モード動作 | 書く／添削／対話 それぞれで想定通り動く |

反復上限：Phase 2→4 のループは最大 2 周。2 周後も不通過の項があれば、L14 境界に明記した上で交付する（無限の磨きよりも誠実な限界が優る）。

---

### Phase 5: 双 Agent 精錬

Phase 4 通過後、2 つの子 agent を並行起動して精錬を行う：

**Agent A（構造評価）**：14 層記述の明確性・実行可能性・エッジケース網羅を評価
**Agent B（憑依検証）**：「AI は作家になっているか、作家について語っているか」を 3 サンプルで判定

主 agent が両レポートを統合し、競合しない改善点を適用する。変更要約をユーザーに提示して確認を取る。

## 既存 skill の更新

ユーザーが「○○の skill を更新して」「新作が出た」と言ったとき：

1. 現 `SKILL.md` から調査日を取得
2. 新規作品のみ解析（`sources/works/new/` に配置）
3. 既存の L1-L3 統計と差分を計算（文体変遷の検出）
4. L10 主題・L13 反パターンに変化があれば差分を追記
5. 全面書き換えではなく増量更新

## 品味守則（速見）

判断に迷ったら戻ってくる指針：

| 原則 | 一言 |
|---|---|
| 作品 > 評論 | 作家本人の 1 行が、評論家の 10 ページに勝つ |
| 統計 > 印象 | 「叙情的」より「文末の体言止め率 18%」 |
| 反パターン > パターン | やらないことが、その人を定義する |
| 誠実な限界 > 完璧な見かけ | L14 の正直さが、skill の信頼を支える |

### 絶対にやらないこと

- 作品の長大な引用を SKILL.md に含める（著作権への配慮）
- L12 で人格診断を断定する（「臨床的診断ではない」の注記必須）
- 生存作家への批判的記述（名誉毀損リスク）
- 情報不足のまま強引に生成する

## 特殊場面

### 生存作家と故人

- **生存作家**：L12 は Big Five のみ。DSM-5 関連は公開医療情報のみ参照。批判的記述は避ける
- **故人**：資料は安定しているが伝記バイアスに注意。複数資料で交差検証する

### 作品集量の不足（10 万字未満）

1. Phase 0.5 の時点で警告：「作品集量が 10 万字未満です。skill の信頼性は限定的になります」
2. L1-L3 の統計値に「サンプル不足」と注記
3. L14 境界に明示

### 非日本語作家

- 原文言語の作品集を使う（翻訳者の文体混入を避けるため）
- L3 音韻・L1 語彙は原文言語の特性で抽出する
- skill の動作言語はユーザーに選択させる（原語／日本語／他）

### 詩人・俳人・歌人

- L9 物語構造はスキップまたは最小化
- L3 音韻・L7 余白・L11 象徴を重点
- 俳人・歌人は L7 に「定型・季語・切れ」の項目を追加

### 個人ブロガー・SNS ライター

- Wikipedia が存在しない場合は本人サイトの About や SNS プロフィールで代替
- L1 口語・L3 句読点癖・L6 読者距離を重点
- L9 物語はスキップ（断片的文体が多いため）

### ユーザー自身を蒸留

1. ユーザー提供の素材（ブログ・過去のメール・SNS 投稿）が作品集になる
2. Wikipedia 代わりにユーザーの自己紹介を使う
3. 自己認知バイアスに注意——可能なら第三者からの評価も補助入力にする
